How i decided to design XPontus XML Editor architecture, to code ...

I needed to use a kind of MVC architecture to maintain easily the application. I decided to create my own “mini-object-container” which configure actions, toolbar and the menubar from a XML file.

I followed some patterns such as singleton pattern, observer pattern,etc.

I am using Netbeans to code most of the gui because it doesn't really affect the code performance. When i first started i was using a kind of notepad but it was taking me too much time to type :-)

Syntax highlighting in java is a real pain for one man, believe it, so i used the modified jedit syntax package from bluej.
